# Task 2: OIDC Authentication Configuration

## Status: 🔄 READY FOR IMPLEMENTATION

## Objective
Configure Tailscale tsidp OIDC authentication for Vikunja.

## Prerequisites
- tsidp running at: https://idp-1.tailf15a72.ts.net/
- Client registered in tsidp with generated client_id and client_secret
- Base infrastructure from Task 1 deployed

## Files to Modify
1. `vikunja/vikunja.yaml` - Add oauth2-proxy container
2. `vikunja/init.sh` - Update to verify OIDC secret

## What Needs to Be Added

### K8s Secret (vikunja-oidc)
Create secret with:
```bash
kubectl create secret generic vikunja-oidc \
  --namespace=vikunja \
  --from-literal=OIDC_CLIENT_ID='<tsidp-client-id>' \
  --from-literal=OIDC_CLIENT_SECRET='<tsidp-secret>' \
  --from-literal=OIDC_ISSUER_URL='https://idp-1.tailf15a72.ts.net/' \
  --from-literal=COOKIE_SECRET='$(openssl rand -base64 32 | tr -d '\n')'
```

### oauth2-proxy Container to Add
Configuration:
- Image: quay.io/oauth2-proxy/oauth2-proxy:v7.6.0
- Provider: oidc
- OIDC issuer URL: https://idp-1.tailf15a72.ts.net/
- Redirect URL: https://vikunja.DOMAIN/oauth2/callback
- Upstream: http://127.0.0.1:3456 (Vikunja API)
- Email domain: * (allow all)
- Pass headers: X-Forwarded-Email, X-Forwarded-User

### Ingress Update
- Change Service target from vikunja to oauth2-proxy (or nginx)
- Vikunja service becomes internal-only

## Implementation Notes

Two approaches:
1. **Simple**: Replace Vikunja service with oauth2-proxy in ingress
2. **Filebrowser-style**: Add nginx proxy for health checks and routing

For Vikunja, approach 1 is simpler since Vikunja has its own health endpoints.
