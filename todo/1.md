# Task 1: Core Infrastructure Setup

## Status: ✅ COMPLETED

## Objective
Create the basic Kubernetes infrastructure for Vikunja deployment.

## Files Created
- ✅ `vikunja/vikunja.yaml` - Namespace, ConfigMap, Deployment, Service, Ingress
- ✅ `vikunja/init.sh` - Local deployment script

## What Was Implemented

### Namespace
- Name: `vikunja`
- Labels: `app=vikunja`

### Storage (hostPath like Filebrowser)
- Database: `/mnt/external/vikunja-db` (hostPath volume)
- Files: `/mnt/external/vikunja-files` (hostPath volume)
- Uses same external disk as filebrowser
- No PVC needed - direct hostPath mounting

### ConfigMap (vikunja-config)
- SQLite database configuration
- Service settings with DOMAIN placeholder
- File storage paths
- Auth configuration (local enabled, OIDC ready)

### Deployment
- Vikunja container with health checks
- Init container for permissions
- Resource limits configured
- Security context set (non-root user)

### Service & Ingress
- ClusterIP service on port 80
- Traefik ingress with TLS
- Domain placeholder: vikunja.DOMAIN
